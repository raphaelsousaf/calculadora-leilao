  <action>
    Em `src/lib/pdf.js`:

    1. Adicionar import: `import { buildSchedule } from './schedule'`.

    2. Dentro do array `rows` em `buildPDF`, **inserir condicionalmente** uma linha de margem **antes** da linha 'Total geral' (per D-24). Calcular margem inline:
       ```js
       const revenda = Number(calc.revendaEsperada) || 0
       const totalEfetivo = (calc.bid || 0) + (calc.commission || 0) + (calc.surety || 0)
       const margem = revenda - totalEfetivo
       const margemPct = revenda > 0 ? (margem / revenda) * 100 : 0
       ```
       Se `revenda > 0`, adicionar entre 'Total geral' e o que vinha antes (i.e., logo após a linha de Parcelas, antes do separador implícito):
       ```js
       if (revenda > 0) {
         rows.splice(rows.length - 1, 0, [
           'Margem estimada',
           `${brl(margem)} (${pct(margemPct)})`,
           true,
           'Revenda esperada − (arremate + comissão + carta de fiança)',
         ])
       }
       ```
       (rows já é o array existente; manter ordem: ...Parcelas, Margem estimada (se revenda > 0), Total geral.)

    3. **Após** o bloco existente de `Documento gerado em ...` (i.e., depois do footer atual), adicionar **antes** do `return doc`:
       ```js
       // Cronograma (per D-24): só se data leilão preenchida
       if (meta?.dataLeilao && (calc.installments || 0) > 0) {
         const schedule = buildSchedule(
           calc.installments,
           meta.dataLeilao,
           Number(calc.intervaloDias) || 30,
           calc.installment || 0,
         )
         if (schedule.length > 0) {
           // Nova página dedicada para evitar overflow ambíguo
           doc.addPage()
           let yy = 56
           doc.setFont('helvetica', 'bold'); doc.setFontSize(13); doc.setTextColor('#1d1d1f')
           doc.text('Cronograma de pagamento', M, yy); yy += 8
           doc.setFont('helvetica', 'normal'); doc.setFontSize(9); doc.setTextColor('#6e6e73')
           const intervaloLabel = Number(calc.intervaloDias) || 30
           yy += 12
           doc.text(`${schedule.length} parcelas · intervalo de ${intervaloLabel} dias · estimado a partir de ${formatDateBR(meta.dataLeilao)}`, M, yy)
           yy += 18

           // Header da tabela
           doc.setDrawColor('#e8e8ed'); doc.line(M, yy, W - M, yy); yy += 14
           doc.setFont('helvetica', 'bold'); doc.setFontSize(9); doc.setTextColor('#6e6e73')
           doc.text('#', M, yy)
           doc.text('Vencimento', M + 60, yy)
           doc.text('Valor', W - M, yy, { align: 'right' })
           yy += 6
           doc.line(M, yy, W - M, yy); yy += 14

           // Linhas (font-size 9pt para caber 30 linhas em 1 página)
           doc.setFont('helvetica', 'normal'); doc.setFontSize(9); doc.setTextColor('#1d1d1f')
           const pageH = doc.internal.pageSize.getHeight()
           schedule.forEach((p) => {
             if (yy > pageH - 60) {
               doc.addPage()
               yy = 56
             }
             doc.text(String(p.n), M, yy)
             doc.text(formatDateBR(p.dueDate), M + 60, yy)
             doc.text(brl(p.amount), W - M, yy, { align: 'right' })
             yy += 14
           })
         }
       }
       ```

    Não alterar a ordem das colunas existentes do card principal. Não duplicar imports de `formatDateBR` (já existe em `pdf.js`).
